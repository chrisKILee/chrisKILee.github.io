Introduction

The motor modules are :
  • info
  • status
  • hvac
  • drive
  • security
  • application
  • traffic
Access to the modules is provided through the interfaces :
  • motor.info
  • motor.status
  • motor.hvac
  • motor.drive
  • motor.security
  • motor.application
  • motor.traffic

Interfaces

MotorObject

Defines the motor interface as part of the window global object.
The motor interface will be always available within the Window object in the ECMAScript hierarchy.
readonly attribute Motor motor

Motor

Root interface.
This is the Automotive root interface that is part of the ECMAScript global object as a property as specified by the MotorObject interface.

MotorAPIError

Generic error interface.
This interface will be used by the APIs in order to throw errors synchronously or return them in the error callbacks of asynchronous methods.
The error codes defined in this interface the ones defined by the DOM Level 3 specification with some additions.

APIs may extend the list of error codes with module-specific ones through the use of the [Supplemental] WebIDL keyword. API specific error numbers start from 100 in order to avoid collision with future DOM Error codes

ErrorCallback

Generic error callback interface.
This callback may be used by those functions that only require an error as input parameter in the error callback.
void onerror(MotorAPIError error)
Method that is invoked when the error occurs.

Features

When the feature
  • http://hyundai.com/api/motor
or any of the features hierarchically under that feature is successfully requested, the interface Motor is instantiated, and the resulting object appears in the global namespace as window.motor.

This is the list of URIs used to declare this API's features, for use in the widget config.xml. For each URL, the list of functions covered is provided.
