AI 시대 개발 전략 로드맵

지금 당장 우리가 해야 할 일

AI 시대의 방향성은 분명하지만, 단번에 도달할 수는 없습니다.
"지금 당장 가능한 것"과 "중간 단계에서 우리가 제공할 것"을 정의합니다.

1. 스펙 주도 개발(SDD)이 당장 완성되지 못하는 이유

SDD 기반 개발 방식은 우리가 도달하려는 차세대 프레임워크의 핵심 지향점입니다. 현재 R&D는 계획된 LEAF 구조, SDD, 테스트 기반 AI 개발 흐름을 점진적으로 고도화하고 있습니다.

그러나 현실적으로 AI 시대는 이제 막 시작되었고, 기술·협업·도구·프로토콜이 매우 빠르게 변화하고 있습니다. 즉, 아직 안정적인 기반이 형성되기 전이며, 바로 이상적인 SDD를 완성하기에는 시간이 필요합니다.

💬 코멘트: 왜 지금 당장 완전체를 요구하면 안 되는가에 대한 이해가 필요합니다.

2. 지금 당장 우리가 해야 할 일 — “환승 전략”

우리는 기술이 성숙하기만을 기다릴 수 없습니다. 사업도, 개발도, 제품도 “지금” 만들어야 합니다.

전략 핵심

최종 목적지까지 한 번에 갈 수 없다면, 여러 차례 환승하면서 최대한 가까운 지점까지 이동한다.

1 현재 단계

일단 사람이 해야 할 것들을 우리가 도구로 최대한 줄여줍니다.

2 미래 단계

이후에는 그 도구를 AI에게 쥐여주어 AI가 사람 대신 개발하도록 만듭니다.

3. 현실 인식과 중재안

지금은 “사람이 직접 해야 하는 일”이 많습니다. AI 생산성에만 의존하면 품질과 속도 모두 불안정해질 수 있습니다. 따라서 당장은 다음의 2가지가 필수적입니다.

사람의 역할

스펙과 구조를 명확히 정의

도구의 역할

반복 노동을 최소화

이것이 우리가 제공할 “중간 단계 도구”의 핵심 역할입니다.

4. 기존 서비스 즉시 개선점

4-1. 로깅 개선

  • 클라이언트 + 서버 로그 통합 수집
  • 에러/행동 패턴 분석 가능
  • 모니터링 화면 제공

현재 개발 방식을 유지하면서 운영을 획기적으로 개선합니다. (로그 = AI 시대 핵심 데이터)

4-2. 서버 개발 지원

기존 서비스에 새로운 기능을 붙이기 위해 추가 서버가 필요할 경우, 이를 위한 인프라와 서버 구축을 대신 지원합니다.

5. 시작점과 인프라 — Backstage 중심

Backstage는 “개발자가 필요 서비스를 선택해 구매하는 키오스크”입니다.

1. 선택

기술 스택, 옵션, 프로퍼티, 인증/스토리지 설정 선택

2. 자동화

서버 구조 생성, GitHub 연결, 인프라 프로비저닝

3. 시작

개발자는 생성된 저장소를 Clone하여 즉시 개발 시작

개발 도구 & 유틸리티

CLI 기능

  • 테이블 생성
  • CRUD API 자동 생성
  • (AI) 유닛 테스트 생성
  • (AI) E2E 테스트 생성

유틸리티 서비스

인증 이메일/SMS 스토리지 이미지 리사이징 푸시 알림 웹소켓 옵저버빌리티

프론트엔드 도구

  • Primitive UI 컴포넌트
  • 실험용 Playground
  • Complex 컴포넌트
  • Figma 연동 디자인 시스템

기존 프레임워크 대체 및 개발 방법의 변화

OutSystems 수준의 완전 자동화를 단기간에 목표로 하지 않습니다. (다윈님 10단계 이론 참고)
우리의 목표는 "편리함"을 제공하되, 최종적으로 AI가 작업할 수 있는 구조를 만드는 것입니다.

AS-IS (OutSystems 등)

  • • 빠른 화면 구성, 쉬운 CRUD, 코드 생성
  • • 확장성/복잡성 한계
  • • 커스텀 로직 구현 어려움
  • • 장기 유지보수 비용 증가

AS-IS (현재 SIS 프레임워크)

  • • 개발 자유도 높음, 표준 기술 기반
  • • 여전히 사람의 작성 비중이 높음
  • • AI가 보조하지만 전체 자동화는 불가

다시 한번, "중간 과정"임을 강조합니다

우리는 AI 개발 시대의 시작점에 있습니다.
여러분이 지금 사용하게 될 도구들은 "우리가 만든 도구를 AI에게 쥐어주는 시대"로 가기 위한 가장 중요한 징검다리입니다.
